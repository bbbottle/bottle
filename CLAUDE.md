## Repository Overview

This is a pnpm monorepo for [bbki.ng](https://bbki.ng), a personal blog and related projects. It uses Turborepo for build orchestration and Changesets for versioning.

## Package Structure

```
packages/
├── site/           # @bbki.ng/site - Main blog (React 18 + Vite 5 + TailwindCSS v4)
├── components/     # @bbki.ng/components - React component library
├── backend/        # @bbki.ng/backend - Cloudflare Workers API (Hono + WebAuthn)
├── cli/            # @bbki.ng/cli - CLI tool for managing content
├── stylebase/      # @bbki.ng/stylebase - TailwindCSS v4 configuration
├── eslint-config/  # @bbki.ng/eslint-config - ESLint configuration
└── config/         # @bbki.ng/config - Shared configuration placeholder
```

## Common Commands

### Root-level Commands

```bash
pnpm install      # Install all dependencies
pnpm dev          # Start dev mode for all packages (turborepo)
pnpm build        # Build all packages (turborepo)
pnpm changeset    # Create a changeset for versioning
pnpm release      # Version packages and publish to npm
```

### Package-Specific Commands

**Site package (`packages/site/`):**

```bash
pnpm dev          # Start Vite dev server with --host
pnpm build        # Type-check and build for production
pnpm serve        # Preview production build
pnpm test         # Run Jest tests
pnpm test:cov     # Run tests with coverage
```

**Backend package (`packages/backend/`):**

```bash
pnpm dev          # Start Wrangler dev server
pnpm deploy       # Deploy to Cloudflare Workers (--minify)
pnpm cf-typegen   # Generate Cloudflare types from wrangler config
```

**Components package (`packages/components/`):**

```bash
pnpm dev          # Start Vite dev server (port 3000)
pnpm build        # Build library with type declarations
pnpm prepare      # Runs build (triggered on install)
```

**CLI package (`packages/cli/`):**

```bash
pnpm dev          # Watch mode TypeScript compilation
pnpm build        # Compile TypeScript
pnpm prepare      # Runs build (triggered on install)
```

**CLI Commands (after build):**

```bash
bbking login                      # GitHub OAuth login
bbking post list                 # List all posts
bbking post add --title <t>      # Create new post
bbking stream list               # List stream entries
bbking stream add --content <c>  # Add stream entry
```

## Architecture

### Site Package (`@bbki.ng/site`)

The main blog application built with React and Vite.

**Key architectural patterns:**

- **Routing**: React Router v6 with nested routes defined in `src/blog/app.tsx`
- **MDX Processing**: Custom MDX pipeline with remark/rehype plugins for frontmatter, TOC, syntax highlighting, and GitHub-flavored markdown
- **State Management**: SWR for server state, React Context for global UI state (`src/blog/context/`)
- **Path Aliases**: `@/` maps to `src/blog/`
- **PWA**: vite-plugin-pwa with custom service worker caching strategies for OSS resources and fonts

**Main entry points:**

- `src/blog/app.tsx` - Root component with route definitions
- `src/blog/index.tsx` - Application entry point
- `vite.config.js` - Extensive Vite configuration with MDX, PWA, TailwindCSS v4, GLSL shaders

### Components Package (`@bbki.ng/components`)

Reusable React component library built with Vite library mode.

**Key architectural patterns:**

- **Library Mode**: Vite builds to ES modules with `preserveModules: true` for tree-shaking
- **Export Strategy**: Conditional exports for development (`lib/`) and production (`dist/`)
- **Component Structure**: Each component in its own folder with co-located stories (`*.stories.tsx`)
- **Styling**: Uses Radix UI primitives, class-variance-authority for variants, TailwindCSS
- **Path Aliases**: `@/` maps to `lib/`

**Build output:**

- Each component is built separately to `dist/` with its own `.mjs` file
- Type declarations generated by `vite-plugin-dts`

### Backend Package (`@bbki.ng/backend`)

Cloudflare Workers API using Hono framework.

**Key architectural patterns:**

- **Framework**: Hono with TypeScript
- **Routes**: Defined in `src/routes/` and mounted in `src/index.ts`
- **Controllers**: Request handlers organized by feature in `src/controllers/`
- **Database**: Cloudflare D1 (SQLite) via `DB` binding
- **Caching**: Cloudflare KV via `KV` binding
- **Authentication**: WebAuthn/Passkeys using `@simplewebauthn/server`

**Configuration:**

- `wrangler.jsonc` - Worker configuration with D1 and KV bindings
- `src/config/app.config.ts` - Hono app with CORS configuration

### CLI Package (`@bbki.ng/cli`)

CLI tool for managing bbki.ng content, built with Commander.js.

**Key architectural patterns:**

- **Framework**: Commander.js for CLI interface
- **Commands**: Organized by feature (post, stream, login) in `src/commands/`
- **Binary**: `bbking` command defined in `package.json`
- **Usage**: Used in CI/CD to add deployment messages to stream

**Build output:**

- Compiled to `dist/cli.js` for distribution

## Testing

- **Framework**: Jest with ts-jest preset
- **Environment**: jsdom
- **Coverage**: Configured for `src/utils/index.ts` only
- **Location**: Tests in `src/blog/__test__/` directories

**Running tests:**

```bash
cd packages/site
pnpm test         # Run all tests
pnpm test:cov     # Run with coverage
pnpm test -- path/to/test.ts  # Run a single test file
pnpm test -- --testNamePattern="test name"  # Run tests matching pattern
```

## Versioning and Publishing

Uses Changesets for automated versioning:

1. `pnpm changeset` - Create a changeset describing your changes
2. `pnpm release` - Version packages and publish to npm

Changesets configuration in `.changeset/config.json`:

- Commits changesets automatically
- Publishes to npm with public access
- Base branch: `main`

## Code Style

- **Formatter**: Prettier
- **Pre-commit**: Husky runs `pnpm pretty-quick --staged`
- **Module System**: ES modules (`"type": "module"` in all packages)
- **TypeScript**: Version 4.x (site, components)

## CI/CD

**Release workflow** (`.github/workflows/release.yml`):

- Triggered on push to `main`
- Creates version PR or publishes to npm via Changesets
- Builds and deploys site to Cloudflare Pages
- Auto-merges version PRs

**Auto-release workflow** (`.github/workflows/autorelease.yml`):

- Triggered on push to `autorelease` branch
- Creates and merges PR from `autorelease` to `main`
